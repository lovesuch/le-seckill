# 秒杀活动 PRD 文档

## 1. 背景与目标
随着市场竞争的加剧，为了提高用户粘性和销售额，公司决定推出秒杀活动。此次秒杀活动将通过微信现金红包和自建微商城优惠券的形式进行，以吸引更多的用户参与。

## 2. 用户故事 / 使用场景

### 用户故事 1: 用户参与秒杀活动
- **描述**: 用户在微信中看到秒杀活动的宣传信息后，点击进入活动页面。
- **使用场景**: 用户通过微信浏览器或小程序访问秒杀活动页面。

### 用户故事 2: 用户领取微信现金红包
- **描述**: 用户成功参与秒杀活动后，系统会立即发放微信现金红包到用户的微信钱包。
- **使用场景**: 用户在活动页面点击领取红包按钮。

### 用户故事 3: 用户领取微商城优惠券
- **描述**: 用户成功参与秒杀活动后，系统会异步将微商城优惠券发放至用户券包。
- **使用场景**: 用户在活动页面点击领取优惠券按钮。

## 3. 功能需求（按优先级P0/P1/P2分类）

### P0: 基本功能
- **功能描述**: 实现秒杀活动的微信现金红包和微商城优惠券发放。
  - **技术实现**:
    - 使用微信支付API发放现金红包。
    - 使用自建微商城系统发放优惠券。

### P1: 高并发处理
- **功能描述**: 确保在高并发情况下，秒杀活动的系统能够稳定运行，不出现崩溃或数据丢失。
  - **技术实现**:
    - 使用Redis进行限流和缓存。
    - 使用MyBatis Plus的批量操作和逻辑删除。

### P2: 用户唯一性限制
- **功能描述**: 确保每个微信用户仅参加一次秒杀活动，避免重复参与。
  - **技术实现**:
    - 在数据库中记录用户的参与状态，并在用户再次尝试参与时返回错误信息。

## 4. 非功能需求（性能/安全/兼容性）

### 性能
- **目标**: 系统能够支持每秒数千次的请求，确保用户体验流畅。
- **技术实现**:
  - 使用SSE实现实时通知，减少服务器压力。
  - 使用Redis进行缓存和限流。

### 安全
- **目标**: 确保用户数据的安全性，防止数据泄露。
- **技术实现**:
  - 使用JWT Token进行认证和授权。
  - 实现CORS和XSS防护。

### 兼容性
- **目标**: 系统需要兼容微信浏览器、小程序等不同平台。
- **技术实现**:
  - 使用响应式设计，确保页面在不同设备上都能正常显示。

## 5. 数据模型概要

### 用户参与状态表 (user_participation)
| 字段名        | 类型         | 描述                 |
|---------------|--------------|----------------------|
| id            | BIGINT       | 主键，自增           |
| user_id       | BIGINT       | 用户ID               |
| activity_id   | BIGINT       | 活动ID               |
| participation_time | DATETIME     | 参与时间             |
| deleted       | TINYINT      | 逻辑删除标志         |

### 微信现金红包表 (wechat_red_packets)
| 字段名        | 类型         | 描述                 |
|---------------|--------------|----------------------|
| id            | BIGINT       | 主键，自增           |
| user_id       | BIGINT       | 用户ID               |
| amount        | DECIMAL(10,2)| 红包金额             |
| status        | VARCHAR(50)  | 状态（已发放、未发放）|
| created_at    | DATETIME     | 创建时间             |

### 微商城优惠券表 (mall_coupons)
| 字段名        | 类型         | 描述                 |
|---------------|--------------|----------------------|
| id            | BIGINT       | 主键，自增           |
| user_id       | BIGINT       | 用户ID               |
| coupon_code   | VARCHAR(50)  | 优惠券代码           |
| status        | VARCHAR(50)  | 状态（已发放、未发放）|
| created_at    | DATETIME     | 创建时间             |

## 6. 验收标准

### 功能验收
- 用户能够成功参与秒杀活动。
- 微信现金红包和微商城优惠券能够正确发放。

### 性能验收
- 系统在高并发情况下能够稳定运行，无崩溃或数据丢失。

### 安全验收
- 用户数据安全，防止数据泄露。
